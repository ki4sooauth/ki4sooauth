mobe01:用户获取账单信息接口（Gooagoo服务器）
功能说明： 用户获取账单信息接口（Gooagoo服务器）
程序映射地址：
返回值例子： mobe01.xml | mobe01.txt
参数说明：
CODE NAME 是否必填 备注说明
userid 用户id 必填 用户编号，唯一，通过UUID产生
sessionid sessionid值 必填 手机登录后的sessionid值
mac手机mac地址选填
mver手机程序版本选填
lidlid选填
entityposition实体店编号选填用户当前所在实体店编号
ctimestamp 时间戳 选填 当时间戳不为空时，返回平台上所有大于时间戳的、已删除的账单编号，并记录返回信息的总条数是否大于“每页显示信息条数（pagesize）“，Y/N
shopid 商家编号 选填
shoptypeid 商家类型编号 选填
deliverystatus取货状态 选填, Y-已取货、N-未取货
orderid 账单编号 必填 根据此编号去查询大于或小于此编号的账单列表。
pagetype 翻页类型 必填 统一分页规则： p-上一页（查询新数据） ，ID传当页第一条数据，平台查询此ID上一页数据 N-下一页（查询旧数据），ID传当页最后一条数据，平台查询此ID下一页数据 例如： 1.数据正序排列 分页大小为3， 全部数据为123，456，789 当前页数据为456 查询上一页数据：pagetype=P,pageid=4,查询结果为123 查询下一页数据：pagetype=N,pageid=6,查询结果为789 1.数据倒序排列 分页大小为3， 全部数据为987，654，321 当前页数据为654 查询上一页数据：pagetype=P,pageid=6,查询结果为987 查询下一页数据：pagetype=N,pageid=4,查询结果为321
pagesize 每页信息显示条数 必填
date 日期 选填 YYYY-MM-DD，如果传入date则查询当日的账单
ordertype 订单/账单类型 选填 订单状态，0-订单，3-账单
业务逻辑说明： 1.根据userid、sessionid校验登录状态；
2.根据orderid、pagetype、pagesize查询账单信息。
备注： 本接口用于查询“Gooagoo服务器”中的账单列表，返回数据不包含部署在商家服务器中“本店账单”的账单列表。
