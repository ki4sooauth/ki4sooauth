moba08:添加新卡（查询商家列表）
功能说明： 添加新卡（查询商家列表）
程序映射地址：
返回值例子： moba08.xml | moba08.txt
参数说明：
CODE NAME 是否必填 备注说明
userid 用户id 选填 用户编号，唯一，通过UUID产生
sessionid sessionid值 选填 手机登录后的sessionid值
mac手机mac地址选填
mver手机程序版本选填
lidlid选填
entityposition实体店编号选填用户当前所在实体店编号
shopid 商家编号 必填 平台通过此参数返回大于或小于
此"商家编号"的商家。
pagetype 翻页类型 必填 统一分页规则： p-上一页（查询新数据） ，ID传当页第一条数据，平台查询此ID上一页数据 N-下一页（查询旧数据），ID传当页最后一条数据，平台查询此ID下一页数据 例如： 1.数据正序排列 分页大小为3， 全部数据为123，456，789 当前页数据为456 查询上一页数据：pagetype=P,pageid=4,查询结果为123 查询下一页数据：pagetype=N,pageid=6,查询结果为789 1.数据倒序排列 分页大小为3， 全部数据为987，654，321 当前页数据为654 查询上一页数据：pagetype=P,pageid=6,查询结果为987 查询下一页数据：pagetype=N,pageid=4,查询结果为321
pagesize 每页显示信息条数 必填
shoptype 商家类别 选填 shoptype、keyword可同时存在。
keyword 关键字 选填 商家名称模糊查询
业务逻辑说明： 1.根据shopid、pagetype、pagesize查询shop_info所有is_del为"N"的正常营业的商家列表。
2.如果shoptype、和keyword不为空，则综合1条件查询shop_info表，获取最终正常营业的商家列表。
3.封装查询结果，返回数据给手机。
