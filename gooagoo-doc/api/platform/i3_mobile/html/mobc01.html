mobc01:商家吆喝(Gooagoo服务器)
功能说明： 商家吆喝(Gooagoo服务器)
程序映射地址：
返回值例子： mobc01.xml | mobc01.txt
参数说明：
CODE NAME 是否必填 备注说明
userid 用户编号 选填 用户编号
sessionid sessionid值 选填 手机登录后的sessionid值
mac手机mac地址选填
mver手机程序版本选填
lidlid选填
entityposition实体店编号选填用户当前所在实体店编号
ctimestamp 时间戳 选填 当时间戳不为空时，返回平台上所有大于
时间戳的、已删除的吆喝编号，并记录返回信息的
总条数是否大于“每页显示信息条数
（pagesize）“，Y/N
shopid 商家编号 选填
pageid 分页编号 选填 平台通过此参数返回大于或小于此
"吆喝分页编号"的吆喝。
pagetype 翻页类型 选填 统一分页规则： p-上一页（查询新数据） ，ID传当页第一条数据，平台查询此ID上一页数据 N-下一页（查询旧数据），ID传当页最后一条数据，平台查询此ID下一页数据 例如： 1.数据正序排列 分页大小为3， 全部数据为123，456，789 当前页数据为456 查询上一页数据：pagetype=P,pageid=4,查询结果为123 查询下一页数据：pagetype=N,pageid=6,查询结果为789 1.数据倒序排列 分页大小为3， 全部数据为987，654，321 当前页数据为654 查询上一页数据：pagetype=P,pageid=6,查询结果为987 查询下一页数据：pagetype=N,pageid=4,查询结果为321
pagesize 分页大小 选填 默认值为50
type 查询吆喝类型 必填 A-全部商家;B-会员商家;C-关注商家
;D-周边商家;E-系统吆喝;F-本店吆喝
gpsx 维度 选填 当“type-查询吆喝类型”为“D-周边商家”
时此参数必填
gpsy 经度 选填 当“type-查询吆喝类型”为“D-周边商家”
时此参数必填
业务逻辑说明： 1.当“type”不为“E”时校验登录状态（暂时没有系统吆喝，此校验先不用考虑）；
2.平台根据type查询到商家信息，然后在根据商家信息查询吆喝信息。
备注：
、 1.默认查询最新吆喝，当最新吆喝条数大于pageSize时，只返回最新pageSize条；
2.本接口用于查询“Gooagoo服务器”中的吆喝列表，返回数据不包含部署在商家服务器中“本店吆喝”的吆喝列表。
