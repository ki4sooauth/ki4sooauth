moba11:用户申请实体卡电子化
功能说明： 用户申请实体卡电子化
程序映射地址：
返回值例子： moba11.xml | moba11.txt
参数说明：
CODE NAME 是否必填 备注说明
userid 用户id 必填 用户编号，唯一，通过UUID产生
sessionid sessionid值 必填 手机登录后的sessionid值
mac手机mac地址选填
mver手机程序版本选填
lidlid选填
entityposition实体店编号选填用户当前所在实体店编号
shopid 店铺id 必填
mobile 手机号 选填 mobile与idno二选一必填
idno 身份证号 选填
phyno 实体卡号 必填
业务逻辑说明： 1.根据userid、sessionid校验登录状态；
2.根据入参，查询member_of_card表，判断此useid是否已经有了本商家的会员卡.如果有，则提示“用户已经是本商家的会员”，如果没有，继续下一步。
3.将信息添加到convert_apply会员申请表中。
3.返回查询结果true/false。
