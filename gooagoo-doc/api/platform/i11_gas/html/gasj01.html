接口gasj01:餐桌状态
功能说明: 餐桌状态查询
程序映射地址:
返回值例子: gasj01.xml | gasj01.txt
参数说明：
CODE NAME 是否必填 备注说明
mac 店员助理MAC地址 必填
shopuserid 店员登录帐（店员id） 必填 对应表shop_user_info中“user_id”
shopentityid 商家实体店编号 必填
业务逻辑说明： 一.主业务逻辑
1.平台根据shopid、shopentityid查询表shop_table_info，获取desk_type（餐桌类型，如3-8人）、sum（当前类型餐桌总数）、deskurl（桌子类型图片地址）
2.平台根据shopentityid、desk_type（餐桌类型）、is_elimination（是否销号，N-未销号）、is_del(0-未删除)在表user_store_queue中查询出记录数量即是排队数
3.平台根据shopentityid、desk_type查询redis缓存中对应的餐桌空闲数（free）及结账数。
4.组合查询结果返回给店员助理
