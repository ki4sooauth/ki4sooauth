接口gasl06:加菜申请
功能说明: 店员助理提交加菜申请（商家落单，转发器生成商家订单后，可申请加菜）
程序映射地址:
返回值例子: gasl06.xml | gasl06.txt
参数说明：
CODE NAME 是否必填 备注说明
mac 店员助理MAC地址 必填
shopuserid 店员登录账号 必填
shopid 商家编号 必填
shopentityid 实体店编号 必填
tablename 桌号 必填
dietlist 忌口json数组字符串 选填 “dietlist”:[{dietname:”,
”dietid:””}　]
goodsinfo 点餐信息json数组字符串 必填 [ { "goodsid":
"00017STNPL7NIGF1T0000LEIISQNS0DL",
"goodsnum":"2"}, { "goodsid":
"00017STNPL7NIGF1T0000LEIISQNS0DL",
"goodsnum":"20"}]
业务逻辑说明： 1.首先校验用户是否已有商家订单，如果有商家订单，则继续以下步骤；否则提示用户“该用户已结账，需重新下单”。
2.根据入参插入bill_add_info表。
3.将操作结果true-提交成功，false-提交失败，返回店员助理。
